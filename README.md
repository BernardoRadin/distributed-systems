# Distributed Systems

## About the Project

This is an **educational project** created to demonstrate **distributed systems concepts in a simple way**, focusing on image storage across servers.

The project simulates a distributed environment where images are saved between servers. To represent load balancing and distributed storage behavior, **two APIs were simulated**, allowing one server to forward requests to another when storage capacity is full.

The main goal of this project is to provide a practical and simplified understanding of how **distribution between servers** can work in real-world scenarios.

## How It Works

* The system simulates **two independent APIs (servers)**.
* Images are uploaded and stored on a server.
* When one server reaches its storage limit, the request is redirected to the other available server.
* This simulates a basic **distributed storage strategy** and **load distribution mechanism**.

## Features

* Image upload and storage
* Communication between simulated servers
* Automatic request forwarding when a server is full
* Simplified distributed system simulation
* Educational approach for learning distributed systems concepts

## Purpose

This project was developed for **educational purposes**, aiming to demonstrate, in a practical way, concepts such as:

* Distributed systems
* Server communication
* Load distribution
* Request forwarding
* Storage management

Although simplified, the project helps illustrate how systems can distribute responsibilities between different services to improve scalability and resource usage.
